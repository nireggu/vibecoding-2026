## 프로젝트: 단계형 퍼즐 게임

### 1. 프로젝트 개요
- 브라우저에서 실행되는 카드 매칭 퍼즐 게임이다.
- 사용자는 같은 그림 2장을 찾아 짝을 맞춘다.
- 모든 단계를 시간 안에 클리어하면 최종 종료 화면과 폭죽 효과를 볼 수 있다.

### 2. 목표
- 처음 접속한 사용자가 시작 버튼을 눌러 게임을 시작하도록 만든다.
- 단계가 올라갈수록 더 큰 보드로 플레이하게 해 몰입감을 높인다.
- 제한 시간과 종료 화면을 통해 게임의 긴장감과 완성도를 높인다.

### 3. 핵심 사용자 흐름
1. 페이지 진입 시 시작 화면을 본다.
2. `시작하기` 버튼을 누르면 첫 단계가 시작된다.
3. 카드 2장을 뒤집어 같은 그림을 찾는다.
4. 한 단계를 모두 맞추면 다음 단계로 자동 이동한다.
5. 마지막 단계를 시간 안에 클리어하면 종료 화면이 나타난다.
6. 종료 화면에서 다시 시작할 수 있다.

### 4. 게임 규칙
- 각 단계는 1분 제한 시간 안에 완료해야 한다.
- 카드 2장을 선택했을 때 같은 그림이면 점수 10점을 얻는다.
- 다른 그림이면 잠깐 보여준 뒤 다시 뒤집는다.
- 보드 중앙에는 홀수 보드에서만 장식용 이모지가 들어간다.
- 마지막 단계까지 모두 클리어해야 게임 전체가 종료된다.
- 시간 초과 시 게임오버가 된다.

### 5. 단계 구성
- 쉬움: 3x3
- 중간1: 4x4
- 중간2: 5x5
- 어려움: 6x6

### 6. 화면 구성
#### 6-1. 시작 화면
- 게임 시작 전 가장 먼저 보여준다.
- 제목, 간단한 안내 문구, `시작하기` 버튼을 표시한다.

#### 6-2. 게임 화면
- 시도 횟수
- 현재 단계
- 남은 시간
- 점수
- 다시 시작 버튼
- 퍼즐 보드
- 상태 메시지

#### 6-3. 종료 화면
- 마지막 단계를 클리어했을 때 보여준다.
- 축하 메시지와 함께 폭죽 효과를 표시한다.
- 다시 시작 버튼을 제공한다.

### 7. 기능 요구사항
- 카드 섞기 기능이 있어야 한다.
- 클릭한 카드의 짝을 판별할 수 있어야 한다.
- 단계별 보드 크기를 자동으로 바꿔야 한다.
- 단계 클리어 시 다음 단계로 자동 이동해야 한다.
- 마지막 단계 클리어 시 종료 화면을 보여줘야 한다.
- 시간 초과 시 게임오버를 표시해야 한다.
- 다시 시작하면 시작 화면으로 돌아가야 한다.

### 8. UI 요구사항
- 시도 횟수, 단계, 남은 시간, 점수를 한눈에 볼 수 있어야 한다.
- 남은 시간 숫자는 빨간색으로 강조한다.
- 시작 화면과 종료 화면은 오버레이 형태로 나타난다.
- 종료 화면은 폭죽이 보이도록 중앙 팝업 형태로 구성한다.
- 모바일 화면에서도 보드와 안내 문구가 잘 보이도록 반응형으로 구성한다.

### 9. 기술 스택
- HTML
- CSS
- JavaScript

### 10. 구현 형태
- 단일 `index.html` 파일로 구성한다.
- 외부 라이브러리 없이 브라우저에서 바로 실행 가능해야 한다.

### 11. 대상 사용자
- 코딩 초보자
- 간단한 브라우저 게임을 빠르게 체험하고 싶은 사용자

### 12. 완료 기준
- 시작 화면에서 게임을 시작할 수 있다.
- 단계별로 퍼즐이 정상적으로 진행된다.
- 시간 제한이 정상 동작한다.
- 최종 클리어 시 종료 화면과 폭죽 효과가 표시된다.
