#### 1. 퍼즐게임 기능 개선

- 점수 계산
```markdown
현재 폴더 Day01에 index.html이 있어. 이걸 기능 개선할 거야. 
시도 횟수 옆에 같은 그림을 맞출 때마다 점수 계산이 돼서 10점씩 올라가도록 해줘. 계산된 점수는 시도횟수 오른쪽에 나오도록 해줘. 나온 파일은 Day02 폴더에 넣어줘.
파일명은 index.html로 만들어줘
```
- 난이도 조정
```markdown
Day02의 puzzle_game 폴더에 있는 파일을 수정할거야.
게임 난이도 조정할래. 레벨 시스템을 도입해줘.
쉬운 난이도는 3x3, 중간 난이도는 4x4, 5x5, 제일어려운 난이도는 6x6 갯수의 퍼즐을 맞추도록 변경해줘. 
난이도는 콤보박스에서 쉬움, 중간1, 중간2, 어려움 이렇게 선택하도록 만들어줘.
```

##### 4-1. Git 설치

##### 4-2. Github 가입
- [GitHub](https://github.com/) 가입
    - 이메일 계정으로 가입되어 있음
- 리포지토리 생성
    - Repository Name 필수
    - Visibility - Public
    - Readme 추가 권장
    - .gitIgnore 개발하는 언어에 따라 선택(필수!) > 안하면 다운받는 사람들의 환경이 이상해 질 수 있음

##### 4-3. GitHub 리포지토리 클론(소스 업로드)
- 내 컴퓨터에 복사

- 윈도우 탐색기에서 경로 선택 및 폴더 생성

    ```bash
    > git clone https://github.com/nireggu/vibecoding-2026.git
    ```
    - 폴더에 git 폴더(vibecoding-2026) 생성
    - git 폴더에 이전에 만들었던 소스 이동하기
- visual studio code - insider 재기동
    - git 폴더로 재탐색

#### 5. 추가 내용

##### 5-1. AI에게 프로젝트 먼저 분석시키기
- 구현된 프로젝트를 사전 분석
    - 개선점, 문제점, 오류(예외포함) AI에 의해서 발견
    - 새로운 프로젝트를 구현 지시

##### 5-2. Plan Mode
- 바로 코딩하지 않고 먼저 계획을 세운 뒤 진행하는 방식
    - 프롬프트 채팅 창에서 + 클릭 후 목표 선택
    - 목표에 해당하는 프롬프트를 작성

```markdown
먼저 구현하지 말고 계획만 작성해줘.

- 필요한 파일 리스트
- 작업순서
- 필요한 폴더 구성
```

##### 5-3. 작업 단위 쪼개기
- 실수

```markdown
쇼핑몰 만들어줘
```

- 실제 예

```markdown
React로 쇼핑몰 프론트엔드를 만들거야.

- 1단계 : 회원가입 화면 작업
- 2단계 : 로그인 화면 작업
- 3단계 : 상품목록 화면
...

단계별로 진행해줘
```

##### 5-4. 컨텍스트 관리
- 컨텍스트가 계속 증가하면 AI도 이전 내용을 지워버림 > / 압축 혹은 아래와 같은 요청 필요

```markdown
PRD.md를 다시 읽어줘
소스코드 부분을 다시 읽어서 분석해줘
```

##### 5-5. 하네스 엔지니어링
- AI가 만든 프로그램이 제대로 동작하는지 AI가 자동으로 확인해주는 검증 방법

```
AI 코드 생성 > 자동 실행 > 자동 테스트 > 오류 발견 > AI 수정 > 재테스트
```

- PRD.md 작성법

```markdown
현재 프로젝트를 테스트 할 수 있는 테스트 하네스를 진행해줘

조건 
- pytest 사용
- 주요 기능 테스트
- 예외 테스트 100회

AI가 수정할 때마다 자동으로 실행되는 테스트 환경을 만들어줘
```

- 테스트 환경 생성 언어별 테스트 환경 기술
    - Javascript - Vitest, Playwright
    - Python - pytest
    - Java - JUnit
    - C# - xUnit, MSTest, NUnit ...
